# Google Play Store Assets

This folder contains all assets required for Google Play Store listing.

## Required Assets

### 1. App Icon
- **File:** `app-icon.png`
- **Size:** 512 x 512 pixels
- **Format:** PNG or JPEG
- **Max Size:** 1 MB
- **Requirements:**
  - Must meet Google Play design specifications
  - No transparency (use solid background)
  - Should represent your app clearly at small sizes
  - Must be unique and not infringe on trademarks

### 2. Feature Graphic
- **File:** `feature-graphic.png`
- **Size:** 1024 x 500 pixels
- **Format:** PNG or JPEG
- **Requirements:**
  - Used as the banner at the top of your app's Play Store listing
  - Should be visually appealing and represent your app
  - Text should be readable at small sizes
  - No phone frames or device mockups

### 3. Screenshots
- **Folder:** `screenshots/`
- **Minimum:** 2 screenshots
- **Maximum:** 8 screenshots
- **Size:** 
  - Phone: 16:9 or 9:16 aspect ratio
  - Minimum: 320px (shortest side)
  - Maximum: 3840px (longest side)
- **Format:** PNG or JPEG
- **Requirements:**
  - Show actual app screens (not mockups)
  - Should demonstrate key features
  - Must be from the actual app

## Recommended Screenshots

1. **Main Inbox Screen** - Shows message list with classifications
2. **OTP Detection** - Shows an OTP message with copy button
3. **Phishing Warning** - Shows a phishing message with warning badge
4. **Detail Screen** - Shows message details with badges and reasons
5. **Settings Screen** - Shows app settings and options

## Optional Assets

### Phone Screenshots (Tablet)
- Only if you support tablets
- Same requirements as phone screenshots

### Promotional Video
- **Format:** YouTube link
- **Length:** 30 seconds to 2 minutes
- **Requirements:**
  - Must be hosted on YouTube
  - Should demonstrate app functionality

## Design Guidelines

### App Icon Design Tips
- Use simple, recognizable imagery
- Avoid text (it won't be readable at small sizes)
- Use high contrast colors
- Test how it looks at 48x48px (smallest display size)
- Consider using an "SMS" or "shield" icon to represent security/classification

### Feature Graphic Design Tips
- Include your app name: "SMS Classifier"
- Highlight key features: "OTP Detection", "Phishing Protection"
- Use your brand colors
- Keep text minimal and readable
- Consider showing a phone with SMS messages

### Screenshot Tips
- Use actual screenshots from your app
- Add captions/text overlays to highlight features (optional)
- Show the most important features first
- Make sure text is readable
- Use consistent styling across all screenshots

## File Naming Convention

- App Icon: `app-icon.png`
- Feature Graphic: `feature-graphic.png`
- Screenshots: `screenshot-01.png`, `screenshot-02.png`, etc.
